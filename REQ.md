# COMChat — Расширенные требования

## 1. Общее описание

**COMChat** — кроссплатформенное TUI-приложение для взаимодействия с последовательными портами (COM / TTY / USB-Serial) в формате, похожем на современный чат-клиент.

Приложение предназначено для инженеров, разработчиков встроенных систем, тестировщиков и специалистов по аппаратному обеспечению, которым требуется удобный инструмент для интерактивного обмена данными с последовательными устройствами.

Основная идея интерфейса — представление взаимодействия с устройством в виде **диалога**, где:

* входящие данные отображаются как сообщения от устройства
* пользовательские команды отображаются как сообщения пользователя
* команды отправляются через поле ввода

---

# 2. Цели системы

Основные цели приложения:

1. Предоставить удобный TUI-интерфейс для работы с последовательными портами.
2. Упростить отправку и повторное использование команд.
3. Обеспечить удобное логирование взаимодействия.
4. Поддерживать работу с несколькими портами одновременно.
5. Работать одинаково на разных операционных системах.

---

# 3. Поддерживаемые платформы

Приложение должно работать на следующих операционных системах:

* Linux
* macOS
* Windows

Поддержка реализуется через абстракцию последовательных портов:

* Windows: `COMx`
* Linux: `/dev/tty*`
* macOS: `/dev/cu.*` / `/dev/tty.*`

---

# 4. Функциональные требования

## 4.1 Работа с последовательными портами

Приложение должно позволять:

* автоматически обнаруживать доступные последовательные порты
* отображать список доступных портов
* открывать соединение с выбранным портом
* закрывать соединение

### Поддерживаемые параметры подключения

Пользователь должен иметь возможность настроить:

* Baud Rate (например: 9600, 115200, и др.)
* Data bits (5, 6, 7, 8)
* Stop bits (1, 1.5, 2)
* Parity (None, Even, Odd)
* Flow control (None, RTS/CTS, XON/XOFF)
* Timeout

Эти настройки должны сохраняться и повторно использоваться.

---

## 4.2 Чат-интерфейс

Интерфейс должен состоять из двух основных областей:

### 1. Область сообщений

Отображает:

* ответы от COM-порта
* команды пользователя

Сообщения должны:

* отображаться в хронологическом порядке
* визуально различаться (цвет / префикс)
* иметь временную метку (опционально)

Пример:

```
[12:01:10] > status
[12:01:10] < OK
[12:01:11] < Temperature: 32C
```

---

### 2. Поле ввода

Позволяет:

* вводить команды
* отправлять их нажатием Enter

Команды отправляются напрямую в последовательный порт.

---

## 4.3 Echo режим

Приложение должно поддерживать включение и отключение **Echo**.

### Echo включен

Отправленная команда отображается в чате.

### Echo выключен

Отправленная команда не отображается (показываются только ответы устройства).

---

## 4.4 История команд

Приложение должно хранить историю введённых пользователем команд.

Функциональность:

* переход по истории с помощью **Arrow Up**
* переход вперёд **Arrow Down**
* повторная отправка команды
* автодополнение предыдущей команды

История должна сохраняться между запусками приложения.

---

## 4.5 Логирование

Приложение должно позволять сохранять:

1. команды пользователя
2. ответы устройства

Логи должны храниться в отдельных файлах:

```
commands.log
responses.log
```

Требования:

* текстовый формат
* UTF-8
* запись в реальном времени
* возможность указать путь к логам

---

## 4.6 Batch режим

Приложение должно поддерживать выполнение набора команд из файла.

### Формат batch файла

Простой текстовый файл:

```
status
version
reset
```

Каждая строка — отдельная команда.

### Поведение

Batch режим должен:

1. отправлять команды последовательно
2. ожидать ответа
3. записывать результаты в лог
4. поддерживать задержку между командами (опционально)

---

## 4.7 Управление несколькими портами

Приложение должно поддерживать **одновременную работу с несколькими портами**.

### Интерфейс

Каждое подключение отображается как **tab**.

Например:

```
[ COM3 ] [ COM5 ] [ /dev/ttyUSB0 ]
```

Пользователь должен иметь возможность:

* переключаться между вкладками
* открывать новый порт
* закрывать вкладку

---

## 4.8 Сохранение конфигураций

Приложение должно позволять сохранять конфигурации портов.

Конфигурация включает:

* имя конфигурации
* параметры порта
* путь к логам
* настройки echo

Пример:

```
STM32 Debug
Arduino
GPS Module
```
